# Changelog

All notable changes to WhatsNew are recorded here.

This file is maintained by both human contributors and AI coding agents. Add
entries as part of the same change that introduces user-visible behavior,
architecture decisions, migrations, dependency changes, or operational changes.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project uses calendar-dated releases until a versioning policy is
introduced.

## Maintenance Rules

- Keep the newest entries at the top.
- Add work-in-progress entries under `Unreleased`.
- Prefer concise, user- or maintainer-facing descriptions over commit-message
  detail.
- Group entries under `Added`, `Changed`, `Fixed`, `Removed`, `Security`, or
  `Internal`.
- Link pull requests, issues, specs, or ADRs when they exist.
- Move `Unreleased` entries into a dated release section when cutting a release.
- Do not delete historical entries unless correcting an error.

## Unreleased

### Added

- The source URL shown at the top of the in-app article reader (and the
  ad-hoc search "read original" view) is now clickable and opens the
  original article in the system's default browser, via
  `tauri-plugin-opener` / `@tauri-apps/plugin-opener` (`opener:default`
  capability).
- The ad-hoc Search page's query and results now persist in
  `web/src/lib/stores/search.ts` (module-level Svelte stores), so navigating
  to a search result's reader view and back no longer clears the search.
  While a search has results, they're silently re-fetched every 60 seconds
  in the background to keep them current.
- The topic page's top bar now shows the current topic's name, centered
  between the view/sort controls (`ViewControls`) on the left and the
  Refresh button on the right.

### Changed

- App icon (`src-tauri/icons/`) now has a transparent background instead of a
  dark navy rounded square, generated by removing the dark background from
  `source.png` and re-running `tauri icon`.

### Added

- Sidebar resize handle: drag the right edge of the sidebar to resize it
  (180-480px), persisted via `sidebarWidth` in
  `web/src/lib/stores/viewPreferences.ts` (`view-preferences.json` via
  `tauri-plugin-store`).
- "Auto-group topics by category" setting (Settings page, off by default):
  groups sidebar topics under category headers (Technology, Sports, Finance,
  etc.) derived from the same keyword rules as topic icons, now shared via
  `web/src/lib/topicCategory.ts`. Persisted as `autoGroupTopics` in
  `view-preferences.json` via `tauri-plugin-store`. Each group header is a
  collapsible toggle (chevron). At most one group is expanded at a time
  (accordion behavior), persisted as `expandedTopicGroup` (category label or
  `null`) in the same store. Group headers are also drag-and-drop
  reorderable (grip handle, shown on hover); the custom order is persisted
  as `topicGroupOrder`. Groups not yet customized fall back to ordering by
  each category's first-appearance position in the user's topic list, so new
  categories slot in near similarly-themed topics instead of an arbitrary
  fixed position. While the topic filter is active, the sidebar always shows
  a flat filtered list (grouping doesn't make sense over a filtered subset),
  reverting to the grouped/accordion view once the filter is cleared.
- Light/dark mode and three selectable color themes - VS Code, Outlook, and
  Console - chosen from a new "Appearance" section on the Settings page.
  Dark mode (VS Code theme) is the default. Implemented as CSS custom
  property palettes in `app.css` switched via `data-theme`/`data-mode`
  attributes on `<html>`, persisted through a new
  `web/src/lib/stores/theme.ts` store (`theme-preferences.json` via
  `tauri-plugin-store`).

### Fixed

- Article summaries from feeds like Google News RSS, whose `<description>` is
  just an `<a>` link to the title plus a source name, are now dropped instead
  of showing as raw escaped HTML. Feed summaries are also stripped of HTML
  markup and de-duplicated when they repeat their first sentence
  (`whatsnew_core::feeds::parse::clean_summary`).
- Some feeds (notably Google News) double-encode HTML in `<description>`
  (e.g. `&amp;lt;a href=...&amp;gt;`), which previously left literal
  `<a href="...">...</a>` tag text visible in article summaries after a
  single decoding pass. `clean_summary`/`strip_html` now re-decodes leftover
  tag-like text up to a couple more times. Additionally, when a feed entry's
  own link is a `news.google.com` redirect, the original article URL is now
  extracted from the `<a href>` embedded in its description (via the new
  `whatsnew_core::feeds::parse::extract_first_link`/`resolve_link`) and used
  as the article's link, so "read the original" goes straight to the source
  instead of Google's redirect.
- Sidebar topic drag-and-drop reordering didn't fire any HTML5 drag events on
  Windows: WebView2's default `dragDropEnabled: true` installs a native OS
  file-drop handler that swallows in-page `dragstart`/`dragover`/`drop`
  events. Disabled via `dragDropEnabled: false` in `tauri.conf.json`. The
  up/down reorder buttons were unaffected.

### Added

- Articles can now be bookmarked/favorited from a topic's article list, any
  view mode, or the in-app reader, via a new bookmark icon
  (`is_favorite` column added in `0003_article_favorites.sql`). A new
  "Bookmarks" sidebar page (`/favorites`) lists every favorited article
  across all topics, most recent first, with the same toggle to unfavorite.

- Added a "Search" page (sidebar entry) for ad-hoc news search: type any
  query to get live Google News results without creating a topic. Results
  open in a self-contained in-app reader, reusing the existing readable
  content extraction.
- Every topic now gets a per-topic Google News search feed as a catch-all,
  linked alongside any curated directory matches when a topic is created.
  Existing topics with no feeds are self-healed (fallback feed added) the
  next time they refresh, so free-text topics that previously got zero
  articles now surface relevant, recent news.
- Sidebar topics now show a topical icon (lucide-svelte) chosen by keyword
  match against the topic name, falling back to a generic newspaper icon.
- The topic page's empty state now explains *why* no articles are showing:
  first refresh in progress, no feeds linked, per-feed fetch errors, or
  simply no recent matches.
- Added article list view modes (grid, compact list, magazine, headlines) and
  sort options (newest, oldest, most relevant) on the topic page, persisted
  globally via `tauri-plugin-store`. Relevancy is scored by how strongly an
  article's title/summary match the topic's name.
- Created this actively maintained changelog for project history and release
  notes.
- Added a self-contained article reader page that fetches and extracts readable
  article text inside the app.
- Added real WhatsNew app icon artwork and regenerated the Tauri desktop icon
  set.
- Added persistent topic ordering with drag-and-drop and up/down controls in the
  sidebar.
- Show article publish times alongside publish dates in article cards and the
  in-app reader.
- Added a system tray icon with show and quit actions; closing the window now
  hides WhatsNew to the tray instead of exiting.
- Added a popular topic cloud below the topic input for quick topic preselection.
- Kept the article reader back button sticky so it remains available at any
  scroll position.
- Show non-ad article images in the in-app reader when they can be extracted from
  the source article page.
- Show the current article title in the sticky reader toolbar.
- Preserve article image placement relative to text in the in-app reader when
  extracting readable content.
- Added native Windows, macOS, and Linux desktop build coverage and GitHub
  Release binaries for tagged releases.

### Internal

- Changed the release workflow so successful `main` CI runs create run-numbered
  prerelease GitHub Releases with desktop binaries, while `v*.*.*` tags remain
  versioned releases.
- Documented in `AGENTS.md` and `CLAUDE.md` that notable changes must update
  `CHANGELOG.md` in the same change.
- Expanded the Tauri bundle configuration and CI/release workflows to validate
  desktop builds across Windows, macOS, and Linux.
- Made CI desktop builds wait for both Rust and frontend checks to pass before
  running.
- Fixed CI desktop builds to use the npm-installed Tauri CLI instead of assuming
  `cargo tauri` is preinstalled on GitHub runners.

## 2026-06-13

### Added

- Implemented the WhatsNew foundation and MVP desktop app:
  topic management, RSS/Atom/JSON Feed matching and refresh, article browsing,
  retention settings, background refresh, and desktop notification support.
- Added `whatsnew-core`, a Tauri-free Rust crate for models, SQLite
  repositories, feed fetching/parsing, feed discovery, topic/feed matching,
  refresh orchestration, and retention pruning.
- Added the Tauri v2 desktop shell with command adapters, app state, scheduler,
  notifications, and local SQLite migrations.
- Added the SvelteKit frontend with API wrappers, stores, app shell, topic
  screens, article list, and settings screen.
- Added PowerShell and bash setup, dev, test, and build scripts.
- Added GitHub Actions workflows for CI and draft releases.

### Internal

- Documented project architecture, commands, conventions, and known caveats in
  `README.md`, `CLAUDE.md`, and `AGENTS.md`.
